# ProxBot v0.5.0

## Espanol

### Resumen

ProxBot v0.5.0 anade un instalador opcional para desplegar ProxBot en un LXC
Debian desde un host Proxmox VE.

### Objetivo

Facilitar una instalacion reproducible en Proxmox sin usar la API de Proxmox,
sin tokens de Proxmox y sin modificar contenedores existentes.

### Cambios publicados

- Script `scripts/proxmox-lxc-install.sh`.
- Modo `--dry-run` para revisar el plan sin crear nada.
- Modo default y modo advanced.
- Deteccion de Proxmox VE, root, storage, bridge y CT ID libre.
- Creacion conservadora de un LXC Debian nuevo.
- Instalacion de Node.js, npm, git y dependencias dentro del contenedor.
- Preparacion de `.env`, `config.json` y `proxbot.service`.
- Documentacion en `docs/PROXMOX-LXC-INSTALL.md` y
  `docs/PROXMOX-LXC-INSTALL.en.md`.
- Guias de prueba en `docs/TESTING-v0.5.0.md` y
  `docs/TESTING-v0.5.0.en.md`.

### Limitaciones

- Solo pensado para Proxmox VE.
- No usa Proxmox API.
- No usa tokens de Proxmox.
- No borra ni modifica VMs/CTs existentes.
- No arranca el bot por defecto hasta que `.env` este configurado.
- No controla VMs ni contenedores desde Discord.

## English

### Summary

ProxBot v0.5.0 adds an optional installer for deploying ProxBot in a Debian LXC
from a Proxmox VE host.

### Goal

Make Proxmox installation repeatable without using the Proxmox API, without
Proxmox tokens, and without modifying existing containers.

### Released changes

- `scripts/proxmox-lxc-install.sh` script.
- `--dry-run` mode to review the plan without creating anything.
- Default and advanced modes.
- Proxmox VE, root, storage, bridge, and free CT ID detection.
- Conservative creation of a new Debian LXC.
- Node.js, npm, git, and dependency installation inside the container.
- `.env`, `config.json`, and `proxbot.service` preparation.
- Documentation in `docs/PROXMOX-LXC-INSTALL.md` and
  `docs/PROXMOX-LXC-INSTALL.en.md`.
- Testing guides in `docs/TESTING-v0.5.0.md` and
  `docs/TESTING-v0.5.0.en.md`.

### Limitations

- Intended only for Proxmox VE.
- Does not use the Proxmox API.
- Does not use Proxmox tokens.
- Does not delete or modify existing VMs/CTs.
- Does not start the bot by default until `.env` is configured.
- Does not control VMs or containers from Discord.

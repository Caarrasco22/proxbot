# ProxBot v0.7.0

## Espanol

### Resumen

ProxBot v0.7.0 anade un sistema basico de permisos por roles de Discord,
desactivado por defecto.

### Objetivo

Permitir restringir comandos sensibles a usuarios con roles especificos de
administrador, sin base de datos ni dependencias adicionales.

### Cambios publicados

- Utilidad interna `utils/permissions.js`.
- Seccion `permissions` en `config.json` con `enabled`, `adminRoleIds` y
  `protectedCommands`.
- Integracion en `index.js`: comprobacion de permisos antes de ejecutar
  comandos slash y antes de responder botones de panel.
- Validacion en `utils/validateConfig.js` para la seccion `permissions`.
- Documentacion en `docs/PERMISSIONS.md` y `docs/PERMISSIONS.en.md`.

### Limitaciones

- Solo funciona dentro de servidores de Discord (no en DMs).
- Desactivado por defecto.
- Si `enabled=true` y `adminRoleIds` esta vacio, todos los comandos protegidos
  quedan bloqueados.
- No sustituye una estrategia real de seguridad del homelab.
- No protege informacion ya publicada en canales.

## English

### Summary

ProxBot v0.7.0 adds a basic Discord role-based permission system, disabled by
default.

### Goal

Allow restricting sensitive commands to users with specific administrator
roles, without a database or additional dependencies.

### Released Changes

- Internal `utils/permissions.js` utility.
- `permissions` section in `config.json` with `enabled`, `adminRoleIds`, and
  `protectedCommands`.
- Integration in `index.js`: permission check before executing slash commands
  and before responding to panel buttons.
- Validation in `utils/validateConfig.js` for the `permissions` section.
- Documentation in `docs/PERMISSIONS.md` and `docs/PERMISSIONS.en.md`.

### Limitations

- Only works inside Discord servers (not in DMs).
- Disabled by default.
- If `enabled=true` and `adminRoleIds` is empty, all protected commands are
  blocked.
- Does not replace a real homelab security strategy.
- Does not protect information already posted in channels.

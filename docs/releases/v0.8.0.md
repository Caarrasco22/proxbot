# ProxBot v0.8.0

## Espanol

### Resumen

ProxBot v0.8.0 anade una integracion opcional y desactivada por defecto con
Proxmox VE en modo solo lectura.

### Objetivo

Consultar informacion de nodos, VMs y contenedores desde Proxmox VE sin
ejecutar acciones sobre la infraestructura.

### Cambios publicados

- Utilidad interna `utils/proxmox.js`.
- Comando `/proxmox` con opciones `estado`, `nodos` y `recursos`.
- Seccion `integrations.proxmox` en `config.json`.
- Token de Proxmox leido desde variable de entorno (no en config.json).
- Campo `rejectUnauthorized` en config (reservado para futuras versiones; en v0.8.0 la validacion TLS la gestiona Node.js/sistema operativo).
- Timeout configurable con `AbortController`.
- Validacion en `utils/validateConfig.js` para la seccion `integrations`.
- Documentacion en `docs/PROXMOX-READONLY.md`, `docs/PROXMOX-READONLY.en.md`,
  `docs/TESTING-v0.8.0.md` y `docs/TESTING-v0.8.0.en.md`.

### Limitaciones

- Solo lectura: no inicia, detiene, reinicia ni elimina recursos.
- Desactivado por defecto.
- No sincroniza automaticamente con el inventario de ProxBot.
- No escribe archivos locales con datos de Proxmox.
- Requiere token de API valido y acceso de red a Proxmox.
- No usa cache persistente en v0.8.0.

## English

### Summary

ProxBot v0.8.0 adds an optional and disabled-by-default Proxmox VE read-only
integration.

### Goal

Query node, VM, and container information from Proxmox VE without executing
actions against the infrastructure.

### Released Changes

- Internal `utils/proxmox.js` utility.
- `/proxmox` command with `estado`, `nodos`, and `recursos` options.
- `integrations.proxmox` section in `config.json`.
- Proxmox token read from environment variable (not in config.json).
- `rejectUnauthorized` field in config (reserved for future versions; in v0.8.0 TLS validation is handled by Node.js / operating system).
- Configurable timeout with `AbortController`.
- Validation in `utils/validateConfig.js` for the `integrations` section.
- Documentation in `docs/PROXMOX-READONLY.md`, `docs/PROXMOX-READONLY.en.md`,
  `docs/TESTING-v0.8.0.md`, and `docs/TESTING-v0.8.0.en.md`.

### Limitations

- Read-only: does not start, stop, restart, or delete resources.
- Disabled by default.
- Does not automatically sync with ProxBot's inventory.
- Does not write local files with Proxmox data.
- Requires a valid API token and network access to Proxmox.
- No persistent cache in v0.8.0.

### Credits

The initial Proxmox API integration idea was proposed by @t0msly3r in PR #2. The v0.8.0 implementation is redesigned to match ProxBot's current philosophy: optional, disabled by default, read-only, no new dependencies, and no destructive actions.

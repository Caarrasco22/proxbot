# ProxBot v0.2.0

## Espanol

### Nuevas funciones

- Setup guiado con `npm run setup`.
- Instalador basico para Debian/Ubuntu en `scripts/install.sh`.
- Nuevo comando `/status`.
- Timeouts configurables para diagnostico:
  - `diagnostics.portTimeoutMs`
  - `diagnostics.urlTimeoutMs`
- Documentacion bilingue:
  - `README.en.md`
  - `docs/INSTALL.en.md`
- Ejemplos de configuracion en `docs/examples/`.

### Cambios

- `package.json` pasa a `0.2.0`.
- `.gitignore` ignora backups `*.backup-*`.
- `config.example.json` incluye seccion `diagnostics`.
- `docs/CONFIG.md`, `docs/SYSTEMD.md` y `docs/GIT.md` se amplian para la nueva instalacion.

### Actualizar desde v0.1.0

```bash
git pull
npm install
npm run check-config
```

Si quieres usar el setup guiado:

```bash
npm run setup
```

Si usas systemd:

```bash
sudo systemctl restart proxbot
```

Si registraste nuevos comandos:

```bash
npm run deploy
```

### Como probar

```bash
npm run check-config
npm run deploy
npm start
```

En Discord:

```text
/status
/panel
/diagnostico
```

### Limitaciones actuales

- Docker no esta incluido todavia.
- GUI web no incluida.
- Alertas automaticas no incluidas.
- `scripts/install.sh` esta orientado a Debian/Ubuntu.
- El instalador usa `apt install nodejs npm`; algunas distribuciones pueden traer una version antigua de Node.js.

## English

### New Features

- Guided setup with `npm run setup`.
- Basic Debian/Ubuntu installer in `scripts/install.sh`.
- New `/status` command.
- Configurable diagnostics timeouts:
  - `diagnostics.portTimeoutMs`
  - `diagnostics.urlTimeoutMs`
- Bilingual documentation:
  - `README.en.md`
  - `docs/INSTALL.en.md`
- Configuration examples in `docs/examples/`.

### Changes

- `package.json` is now `0.2.0`.
- `.gitignore` ignores `*.backup-*` backups.
- `config.example.json` includes the `diagnostics` section.
- `docs/CONFIG.md`, `docs/SYSTEMD.md`, and `docs/GIT.md` were expanded for the new setup flow.

### Upgrade From v0.1.0

```bash
git pull
npm install
npm run check-config
```

If you want to use the guided setup:

```bash
npm run setup
```

If you use systemd:

```bash
sudo systemctl restart proxbot
```

If slash commands changed:

```bash
npm run deploy
```

### How To Test

```bash
npm run check-config
npm run deploy
npm start
```

In Discord:

```text
/status
/panel
/diagnostico
```

### Current Limitations

- Docker is not included yet.
- Web UI is not included.
- Automatic alerts are not included.
- `scripts/install.sh` is focused on Debian/Ubuntu.
- The installer uses `apt install nodejs npm`; some distributions may ship an old Node.js version.

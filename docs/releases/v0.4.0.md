# ProxBot v0.4.0

## Espanol

### Resumen

ProxBot v0.4.0 se centra en inventario basico del homelab desde Discord.

### Objetivo

Explicar que servicios existen, donde estan y como encontrarlos sin ejecutar
acciones sobre la infraestructura.

### Cambios publicados

- Utilidad interna `utils/inventory.js`.
- Comando `/inventario` con filtros por tag, categoria y busqueda.
- Comando `/servicio-info` para consultar una ficha de servicio.
- Campos opcionales de inventario en `config.json`: `category`, `owner`,
  `location`, `notes` y `tags`.
- Boton `Inventario` en `/panel`.
- Documentacion en `docs/INVENTORY.md` y `docs/INVENTORY.en.md`.

### Limitaciones

- Solo lectura.
- No ejecuta comandos SSH.
- No hace checks de red desde los comandos de inventario.
- No consulta Proxmox, Uptime Kuma ni APIs externas.
- No sincroniza servicios automaticamente.

## English

### Summary

ProxBot v0.4.0 focuses on a basic homelab inventory from Discord.

### Goal

Explain which services exist, where they are, and how to find them without
executing actions against the infrastructure.

### Released changes

- Internal `utils/inventory.js` utility.
- `/inventario` command with tag, category, and search filters.
- `/servicio-info` command for service details.
- Optional inventory fields in `config.json`: `category`, `owner`, `location`,
  `notes`, and `tags`.
- `Inventario` button in `/panel`.
- Documentation in `docs/INVENTORY.md` and `docs/INVENTORY.en.md`.

### Limitations

- Read-only.
- Does not run SSH commands.
- Inventory commands do not run network checks.
- Does not query Proxmox, Uptime Kuma, or external APIs.
- Does not sync services automatically.
